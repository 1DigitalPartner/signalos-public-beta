# SignalOS Community Code of Conduct

## Purpose

The SignalOS public beta community exists to support useful product testing, professional feedback, responsible security reporting, and constructive discussion.

Participation should improve the product without exposing users, repositories, customer information, or connected systems to unnecessary risk.

## Expected behavior

Participants are expected to:

- communicate clearly and respectfully;
- focus criticism on workflows, behavior, evidence, and outcomes;
- provide reproducible details when reporting defects;
- distinguish product feedback from security findings;
- protect confidential and personal information;
- use only assets and repositories they are authorized to test;
- review generated work before approval or merge;
- respect different levels of technical experience;
- avoid misleading claims about SignalOS, TanziTech, or other participants;
- follow the repository license and security policy.

## Unacceptable behavior

The following is not acceptable:

- harassment, threats, discrimination, or personal attacks;
- publishing secrets, tokens, cookies, private keys, or credentials;
- exposing customer data or private repository contents;
- sharing active exploit details before coordinated remediation;
- testing systems without authorization;
- intentionally disrupting the hosted service;
- using the beta to distribute malware or malicious payloads;
- impersonating TanziTech, SignalOS staff, testers, or partners;
- spam, deceptive promotion, or unrelated commercial solicitation;
- submitting fabricated test results or misleading evidence;
- repeatedly ignoring moderator or security instructions.

## Product criticism

Direct and critical feedback is welcome when it is specific and grounded.

Good feedback explains:

- the workflow tested;
- expected behavior;
- actual behavior;
- user or business impact;
- safe evidence;
- a suggested improvement when available.

Negative feedback will not be removed merely because it is unfavorable.

## Security-sensitive content

Do not publish vulnerability details in Issues, Discussions, pull requests, or comments.

Follow `SECURITY.md` and report privately to:

`security@tanzitech.com`

When uncertain, report privately first.

## Confidentiality and privacy

Do not submit:

- personal data that is not necessary;
- private email addresses without permission;
- customer records;
- internal company documents;
- confidential source code;
- private repository names or URLs;
- production identifiers that expose users or infrastructure;
- unredacted screenshots containing sensitive information.

## Enforcement

Repository maintainers may:

- edit or remove sensitive content;
- hide or lock discussions;
- close issues that violate the reporting process;
- remove spam or abusive content;
- restrict or block participation;
- move security reports to a private channel;
- preserve evidence needed for investigation.

Enforcement decisions should prioritize user safety, product integrity, and constructive participation.

## Reporting conduct concerns

Conduct concerns may be reported to:

`info@tanzitech.com`

Use the subject:

`SignalOS Community Conduct Report`

Include only the information needed to assess the concern.

## Scope

This Code of Conduct applies to:

- repository Issues;
- GitHub Discussions;
- pull requests;
- comments;
- beta feedback sessions;
- community spaces operated for SignalOS;
- direct interactions related to the public beta program.

## Final standard

Participate in a way that makes SignalOS safer, clearer, more useful, and easier to evaluate.
