# Contributing to the SignalOS Public Beta Hub

Thank you for helping evaluate SignalOS.

This repository is a public documentation and feedback hub for a commercial TanziTech product. It is not the production source repository.

## Useful contributions

The most valuable contributions include:

- reproducible bug reports;
- structured product feedback;
- completed beta test missions;
- safe GitHub connector evaluations;
- recommendation-quality reviews;
- use cases tied to real business outcomes;
- integration requests with clear controls;
- corrections to public documentation;
- redacted screenshots that clarify a workflow;
- design partner evidence.

## Before contributing

Read:

- `README.md`
- `GETTING_STARTED.md`
- `TESTING_GUIDE.md`
- `FEEDBACK_GUIDE.md`
- `SECURITY.md`
- `CODE_OF_CONDUCT.md`
- `LICENSE`

## Choose the correct channel

Use Issues for:

- reproducible defects;
- structured product feedback;
- connector test results;
- Premium Agent test results;
- feature and integration requests.

Use Discussions for:

- broader beta experiences;
- use cases;
- non-sensitive ideas;
- product questions;
- integration exploration.

Use the private security process for:

- vulnerabilities;
- approval bypasses;
- unauthorized access;
- secret exposure;
- unintended external writes;
- cross-account data exposure.

## Documentation pull requests

Documentation corrections may be submitted through pull requests.

A useful documentation pull request should:

- solve a clear problem;
- remain limited in scope;
- preserve the commercial positioning;
- avoid private implementation details;
- avoid unsupported product claims;
- avoid secrets and personal data;
- update related links when necessary;
- explain why the change improves testing or understanding.

## Contribution license

By submitting a contribution, you agree to the contribution terms in `LICENSE`.

Do not submit material you do not have the right to share.

## Public evidence rules

Evidence must be sanitized.

Remove:

- access tokens;
- cookies;
- private keys;
- customer data;
- private repository contents;
- internal identifiers when unnecessary;
- confidential company information;
- non-public URLs;
- unrelated personal information.

## Quality standard

Contributions should be:

- specific;
- test-based;
- commercially relevant where possible;
- safe to publish;
- written in clear language;
- honest about uncertainty;
- focused on observable behavior or outcomes.

## Review process

Maintainers may:

- request clarification;
- request redaction;
- close duplicates;
- move security-sensitive material to a private process;
- revise wording for consistency;
- decline changes that conflict with product direction;
- incorporate feedback into private product development.

Submission does not guarantee implementation or publication.

## Product source code

The private production source code is not accepted through this repository.

Do not submit attempts to reconstruct, expose, or distribute private SignalOS implementation details.
